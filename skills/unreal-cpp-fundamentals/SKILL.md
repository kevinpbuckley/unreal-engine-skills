# Unreal Engine C++ fundamentals

Unreal C++ is standard C++ plus a **reflection layer** generated by the Unreal Header Tool
(UHT). Getting the reflection macros right is the difference between code that compiles,
shows up in Blueprints, replicates, and survives garbage collection — and code that doesn't.

## When to use this skill

- Creating or editing any `U*`/`A*` class or `F*` struct in C++.
- Exposing a member or function to Blueprints, the editor, or replication.
- Diagnosing UHT/reflection errors ("Unrecognized type", "missing generated.h", etc.).
- Choosing a pointer type (`TObjectPtr`, raw, `TWeakObjectPtr`, `TSharedPtr`).

## Mental model

- **UObject** is the base of the reflected, garbage-collected object system. `UCLASS`,
  `UPROPERTY`, and `UFUNCTION` register types/members with the engine so they can be
  serialized, edited, scripted, networked, and GC-tracked.
- **UHT** parses your headers and generates a `<Name>.generated.h`. You must `#include` it
  **last** and add the `GENERATED_BODY()` macro inside the type.
- The engine **owns the lifetime** of UObjects via the garbage collector. You keep them alive
  by holding them in a `UPROPERTY()`. Plain C++ pointers do **not** keep UObjects alive.

## Class prefixes (mandatory naming)

| Prefix | Meaning | Example |
|---|---|---|
| `U` | UObject-derived (not Actor) | `UActorComponent`, `UMyDataAsset` |
| `A` | Actor-derived (placeable in world) | `AActor`, `AMyCharacter` |
| `F` | plain struct / non-UObject type | `FVector`, `FMyConfig` |
| `E` | enum | `EMyState` |
| `I` | interface class (the `IName` half of a `UName`) | `IInteractable` |
| `T` | template | `TArray`, `TObjectPtr` |
| `S` | Slate widget | `SButton` |

The class name after the prefix must match the file name (minus prefix): `AMyPawn` in `MyPawn.h`.

## Anatomy of a UObject class

```cpp
// MyActor.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "MyActor.generated.h"   // ALWAYS LAST include

UCLASS(Blueprintable, BlueprintType)
class MYGAME_API AMyActor : public AActor   // MYGAME_API = module export macro
{
    GENERATED_BODY()

public:
    AMyActor();

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
    float Health = 100.f;

    UFUNCTION(BlueprintCallable, Category = "Stats")
    void ApplyDamage(float Amount);

protected:
    virtual void BeginPlay() override;

    UPROPERTY(VisibleAnywhere)
    TObjectPtr<USceneComponent> Root;   // UPROPERTY keeps it alive + visible to GC
};
```

```cpp
// MyActor.cpp
#include "MyActor.h"

AMyActor::AMyActor()
{
    PrimaryActorTick.bCanEverTick = true;
    Root = CreateDefaultSubobject<USceneComponent>(TEXT("Root"));
    SetRootComponent(Root);
}

void AMyActor::BeginPlay()
{
    Super::BeginPlay();   // call Super on engine overrides
}

void AMyActor::ApplyDamage(float Amount)
{
    Health = FMath::Max(0.f, Health - Amount);
}
```

Rules that prevent the most common build breaks:
- `#include "X.generated.h"` is the **last** include in the header.
- Every reflected `class`/`struct` has `GENERATED_BODY()` (UObjects/structs) — interfaces use
  `GENERATED_BODY()` in both the `U*` and `I*` parts.
- The class is annotated with the **module API macro** (`MYGAME_API`) so other modules can link it.
- Always call `Super::` in overridden engine virtuals (`BeginPlay`, `Tick`, `EndPlay`, etc.).

## UCLASS specifiers you'll actually use

- `Blueprintable` — can be subclassed by a Blueprint.
- `BlueprintType` — can be used as a variable/type in Blueprints.
- `Abstract` — cannot be instantiated directly.
- `Config=Game` — properties marked `Config` read/write from `*.ini`.
- `meta=(...)` — editor/Blueprint hints (e.g. `meta=(PrioritizeCategories="Stats")`).

## UPROPERTY specifiers (the high-value ones)

Edit/visibility (editor Details panel):
- `EditAnywhere` / `EditDefaultsOnly` (CDO only) / `EditInstanceOnly` (placed instance only).
- `VisibleAnywhere` / `VisibleDefaultsOnly` / `VisibleInstanceOnly` — read-only in editor.

Blueprint access:
- `BlueprintReadWrite` — get/set in BP. `BlueprintReadOnly` — get only.

Common combos & meta:
- `Category = "Group"` — organizes the Details panel (use for everything).
- `meta=(ClampMin="0", ClampMax="100", UIMin="0", UIMax="100")` — value ranges.
- `meta=(AllowPrivateAccess="true")` — expose a `private` member to BP.
- `Transient` — not saved. `SaveGame` — included in SaveGame serialization.
- `Replicated` / `ReplicatedUsing=OnRep_Func` — see `networking-and-replication`.
- `Instanced` — for instanced subobjects (e.g. per-instance configurable objects).

**Memory rule:** any UObject* member you want kept alive must be a `UPROPERTY`. Use
`TObjectPtr<UType>` for members (editor-aware, access-tracked); raw `UType*` is fine for
locals/params. Without `UPROPERTY`, the GC may collect the object and leave a dangling pointer.

## UFUNCTION specifiers

- `BlueprintCallable` — callable from BP graphs (has side effects).
- `BlueprintPure` — no side effects; shows as a pure node (no exec pins).
- `BlueprintImplementableEvent` — declared in C++, **implemented in Blueprint** (no C++ body).
- `BlueprintNativeEvent` — C++ default (`Func_Implementation`) overridable in BP.
- `CallInEditor` — adds a button in the Details panel.
- `Server` / `Client` / `NetMulticast` + `Reliable`/`Unreliable` — RPCs (see replication).
- `Exec` — console command.

`BlueprintNativeEvent` pattern:
```cpp
UFUNCTION(BlueprintNativeEvent, Category="AI")
void OnSpotted(AActor* By);
void OnSpotted_Implementation(AActor* By);   // C++ default impl
```

## USTRUCT, UENUM, UINTERFACE

```cpp
USTRUCT(BlueprintType)
struct FLoadout
{
    GENERATED_BODY()
    UPROPERTY(EditAnywhere, BlueprintReadWrite) int32 Ammo = 0;
};

UENUM(BlueprintType)
enum class EDoorState : uint8 { Closed, Opening, Open, Closing };

// Interface: declare both halves
UINTERFACE(MinimalAPI, Blueprintable)
class UInteractable : public UInterface { GENERATED_BODY() };
class IInteractable
{
    GENERATED_BODY()
public:
    UFUNCTION(BlueprintNativeEvent) void Interact(AActor* Instigator);
};
```
`UENUM(BlueprintType)` enums must be `uint8`-backed to be Blueprint-usable.

## Object construction & helpers

- `CreateDefaultSubobject<T>(TEXT("Name"))` — **constructor only**, for owned components.
- `NewObject<T>(Outer)` — runtime UObject creation (non-Actors).
- `GetWorld()->SpawnActor<T>(Class, Transform)` — spawn actors at runtime.
- `Cast<T>(Obj)` — safe checked cast (returns null on failure); never `static_cast` UObjects.
- `IsValid(Obj)` — true if non-null and not pending kill; prefer over a bare null check.

## Pointer/ownership cheat sheet

| Need | Use |
|---|---|
| UObject member you own/keep alive | `UPROPERTY() TObjectPtr<UType>` |
| Local/parameter UObject reference | raw `UType*` |
| Non-owning ref that may go away | `TWeakObjectPtr<UType>` (check `.Get()`/`.IsValid()`) |
| Non-UObject heap object, shared | `TSharedPtr<T>` / `TSharedRef<T>` |
| Asset reference loaded on demand | `TSoftObjectPtr<T>` / `TSoftClassPtr<T>` |

## Gotchas

- **Forgot `UPROPERTY` on a UObject* member** → random crashes after GC. Most common bug.
- **`generated.h` not last / missing `GENERATED_BODY()`** → cryptic UHT errors.
- **Editing a header's reflected surface** requires recompiling (and sometimes a Live Coding
  full rebuild); structural changes (new UPROPERTY/UFUNCTION) often need a normal build.
- **Calling editor-only API from runtime code** → packaging failures; guard with `WITH_EDITOR`.
- **`FName`/`FString`/`FText` mix-ups** — see `unreal-core-types-and-containers`.
