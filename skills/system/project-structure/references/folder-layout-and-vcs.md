# Folder layout and version control — full reference

Deep dive for [../SKILL.md](../SKILL.md). Covers every folder in a UE project, what is
authored vs generated, content paths and mount points, and a complete `.gitignore`
strategy. Grounded in UE 5.7
(`Engine/Source/Runtime/Core/Public/Misc/Paths.h`,
`Engine/Source/Runtime/Projects/Public/ProjectDescriptor.h`).

Official docs (verified live):
- Unreal Engine Directory Structure — <https://dev.epicgames.com/documentation/unreal-engine/unreal-engine-directory-structure>

## Full folder map

```
MyGame/
├── MyGame.uproject             # JSON project descriptor — COMMIT
│
├── Config/                     # COMMIT
│   ├── DefaultEngine.ini       # Rendering, default maps/GameMode, collision, plugins
│   ├── DefaultGame.ini         # Project identity, Asset Manager, game settings
│   ├── DefaultInput.ini        # Legacy axis/action bindings
│   ├── DefaultEditor.ini       # Editor preferences shipped with the project
│   ├── DefaultGameplayTags.ini # Tag table
│   └── DefaultGameUserSettings.ini  # Rarely committed; often left to Saved/
│
├── Content/                    # COMMIT (binary, use Git LFS or Perforce)
│   └── **/*.uasset, **/*.umap
│
├── Source/                     # COMMIT
│   ├── MyGame.Target.cs        # Game target (TargetType.Game)
│   ├── MyGameEditor.Target.cs  # Editor target (TargetType.Editor)
│   └── MyGame/                 # Primary game module
│       ├── MyGame.Build.cs
│       ├── Public/             # Headers exposed to dependent modules
│       └── Private/            # .cpp files and private headers
│           └── MyGameModule.cpp  # IMPLEMENT_PRIMARY_GAME_MODULE
│
├── Plugins/                    # COMMIT selectively
│   └── MyPlugin/
│       ├── MyPlugin.uplugin    # COMMIT
│       ├── Source/             # COMMIT
│       └── Content/            # COMMIT
│
├── Binaries/           # GENERATED — ignore
│   └── Win64/MyGame*.dll
│
├── Intermediate/       # GENERATED — ignore
│   ├── Build/
│   ├── ProjectFiles/
│   └── ShaderAutogen/
│
├── DerivedDataCache/   # GENERATED — ignore (rebuild on demand)
│
└── Saved/              # GENERATED — ignore
    ├── Config/         # Runtime config overrides (per-machine)
    ├── Logs/
    ├── Autosaves/
    └── StagedBuilds/   # Cooked output from "Package Project"
```

## What each folder contains

### Config/

`.ini` files that control the engine and project. The `Config/` folder is part of the
project descriptor and is always committed. For the cascade and syntax see
[references/config-system.md](config-system.md).

`FPaths::ProjectConfigDir()` resolves to this folder at runtime
(`Runtime/Core/Public/Misc/Paths.h:298`).

### Content/

All Unreal assets (`.uasset`) and maps (`.umap`) live here. The engine refers to them
via the `/Game/` virtual mount point; never use OS paths to reference content.

`FPaths::ProjectContentDir()` resolves to this folder (`Paths.h:291`).

Binary assets do not text-merge. Use **Git LFS** (tracking `*.uasset` and `*.umap`) or
Perforce with exclusive checkout on any project where multiple people edit the same
assets.

### Source/

C++ module source code. The Unreal Build Tool (UBT) discovers modules here by looking
for `*.Build.cs` files. Only the game's C++ source is here; generated headers go to
`Intermediate/`.

A Blueprint-only project has no `Source/` folder. The first time you add a C++ class,
Unreal creates `Source/` and the two Target files.

### Plugins/

Project-local plugins — each a self-contained directory with a `.uplugin` descriptor,
`Source/`, and optionally `Content/`. Commit `Source/` and `Content/`; ignore
`Plugins/**/Binaries/` and `Plugins/**/Intermediate/`.

Engine plugins live in `Engine/Plugins/` and are not duplicated into the project.

### Binaries/

Compiled output: game DLLs (`.dll`/`.so`) and the editor executable recompiled for hot
reload. Fully regenerated by UBT; never commit.

### Intermediate/

Build temporaries: generated reflection headers (`*.generated.h`), UBT build records,
shader compilation cache, IDE project files. Regenerated by UBT or `GenerateProjectFiles`;
never commit.

Deleting `Intermediate/` and rerunning UBT is the standard fix for "stale generated
header" errors.

### DerivedDataCache/ (DDC)

Cooked, transcoded, and derived asset data. The DDC can be shared across a team using a
shared network DDC path (`[InstalledDerivedDataBackend]` in `Engine/Config/BaseEngine.ini`).
Never commit the local DDC.

`FPaths::ProjectSavedDir()` and `FPaths::ProjectIntermediateDir()` resolve to `Saved/` and
`Intermediate/` respectively (`Paths.h:305`, `Paths.h:312`).

### Saved/

Runtime-written files: log files, config overrides (in `Saved/Config/`), autosaves, and
packaged builds (`Saved/StagedBuilds/`). All regeneratable; never commit.

The editor writes project-settings changes to `Saved/Config/...` during editing sessions
and then copies them to `Config/Default*.ini` on save. If you quit without saving,
`Saved/Config/` may contain changes not in `Config/`.

## Content paths and mount points

The engine uses virtual asset paths — never reference assets by OS path. The mount
points are:

| Virtual prefix | Maps to |
|---|---|
| `/Game/` | Project `Content/` |
| `/Engine/` | Engine `Content/` (`Engine/Content/`) |
| `/PluginName/` | Plugin `Content/` |
| `/Script/ModuleName` | C++ classes in that module (not a file path — used in config sections and soft class references) |

An asset's full object path has the form `/Game/Path/To/Asset.Asset` (package path + `.`
+ asset name). Omit the `.AssetName` suffix to get the package path only.

Example references:

```cpp
// Load by path at runtime (async-load preferred; see asset-management):
TSoftObjectPtr<UStaticMesh> MeshRef =
    FSoftObjectPath(TEXT("/Game/Props/Crate.Crate"));

// Hard reference (kept in memory as long as the object exists):
UStaticMesh* M = LoadObject<UStaticMesh>(nullptr,
    TEXT("/Game/Props/Crate.Crate"));
```

## .gitignore for an Unreal project

Minimal, correct `.gitignore`:

```gitignore
# Generated build output
Binaries/
Intermediate/
DerivedDataCache/
Saved/

# Visual Studio
.vs/
*.VC.db
*.VC.opendb

# Rider / CLion
.idea/
*.sln
*.vcxproj
*.vcxproj.filters

# Per-plugin generated dirs
Plugins/**/Binaries/
Plugins/**/Intermediate/

# Optional: ignore cooked content if you cook from source
# Build/
# Saved/StagedBuilds/
```

Commit everything else: `*.uproject`, `Config/`, `Content/`, `Source/`, `Plugins/**`
(minus generated subdirs), `*.uplugin`.

### Git LFS for binary content

Add a `.gitattributes` file to track binaries via LFS:

```
*.uasset filter=lfs diff=lfs merge=lfs -text
*.umap   filter=lfs diff=lfs merge=lfs -text
*.uanim  filter=lfs diff=lfs merge=lfs -text
```

Without LFS (or Perforce), each committed change to a `.uasset` bloats the repository
permanently since Git stores the full binary delta.

## Version notes

- `DerivedDataCache/` used to be placed inside `Engine/` in UE4 when projects were
  always sub-directories of the engine. In UE5 standalone projects, it lives in the
  project root.
- World Partition projects store streaming cell data in `Content/` as regular `.uasset`
  files alongside the level. The folder structure is otherwise identical.
- The `Build/` subdirectory (inside `Source/` or project root) is used by some platform
  SDKs for pre/post-build steps. It is not generated and may be committed if your
  project uses it.
